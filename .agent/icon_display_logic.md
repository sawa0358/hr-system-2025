# システムアイコン・iOSホームアイコンの表示ロジック解説書
# Icon Display Logic Handover Document

このドキュメントは、システム内アイコンおよびiOSホーム画面用アイコンにおいて、**「白枠（余白）などが見えないように全画面表示させるロジック」**を解説するものです。

今後の開発やAIによる改修において、同様のデザインを維持するための参照情報として使用してください。

---

## 1. システム内アイコン (Sidebar Icon)

アプリ内（PC/モバイルブラウザ表示時）のサイドバー左上に表示されるロゴアイコンの表示ロジックです。

### 実装コード
**ファイル**: `src/app/page.tsx` (Sidebarコンポーネント部分)

```tsx
<div className="w-12 h-12 relative rounded-lg overflow-hidden shrink-0">
  <Image
    src="/logo-v4.png"
    alt="Reno View Logo"
    fill
    className="object-cover"
    sizes="48px"
  />
</div>
```

### ロジックのポイント
1.  **コンテナによる切り抜き (`overflow-hidden` + `rounded-lg`)**
    *   親要素（`div`）に `rounded-lg` (CSS: `border-radius`) を適用し、かつ `overflow-hidden` を設定しています。
    *   これにより、正方形の画像であっても角が切り取られ、UIに馴染む形状になります。

2.  **`object-cover` による全領域埋め**
    *   `next/image` コンポーネントに `fill` プロパティ（親要素いっぱいに広げる）と、CSSクラス `object-cover` を指定しています。
    *   **重要**: `object-cover` は、画像の縦横比を維持したまま、コンテナの短い辺に合わせて画像を拡大・トリミングし、コンテナ全体を埋め尽くします。これにより、画像とコンテナのアスペクト比が多少異なっても、上下左右に余白（白枠）が生じることがありません。

## 2. iOSホーム画面アイコン (Mobile App Icon)

iPhone/iPadのSafariで「ホーム画面に追加」を行った際に表示されるアプリアイコンの表示ロジックです。

### 実装方法
**参照ファイル**: `src/app/apple-icon.png`

### ロジックのポイント
1.  **Next.js Metadata API の利用**
    *   `src/app/` ディレクトリ直下に `apple-icon.png` という特定のファイル名で画像を配置しています。
    *   Next.jsはこのファイルを自動検出し、ビルド時に適切な `<link rel="apple-touch-icon" ...>` タグをHTMLヘッダーに生成します。特別なコーディングは不要です。

2.  **Full Bleed（裁ち落とし）デザイン**
    *   **重要**: 配置している `apple-icon.png` 自体が、**「余白なしの正方形（Full Bleed）」**として作成されています。
    *   画像の端まで背景色やグラフィックで塗りつぶされており、透明部分や意図しない白い縁取りが含まれていません。

3.  **iOSの仕様への適合**
    *   iOSは、`apple-touch-icon` として指定された正方形の画像を自動的にシステム標準の形状（スクワークル）にマスク（切り抜き）します。
    *   画像が透明背景の場合、iOSは背景を黒（または白）に塗りつぶしてしまうため、意図しない枠が見える原因になります。本システムでは**不透明な画像を端まで描画**することで、iOSのマスク適用後も隙間なくアイコンが表示されるようにしています。

## まとめ

| 項目 | ポイント |
| :--- | :--- |
| **サイドバーアイコン** | CSSの `object-cover` でコンテナを埋め尽くし、`rounded-lg` で角を落とす。 |
| **iOSアイコン** | `src/app/apple-icon.png` に配置し、画像自体を余白なし・不透明の正方形でデザインする。 |
